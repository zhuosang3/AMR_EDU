# HLS CANopen → ROS2 桥接层 (hls_ros2)

## 目标

在 `hls_canopen` 项目中新增一个 ROS2 桥接二进制 `hls_ros2`，用 **Rust + r2r** 实现差速底盘驱动节点。复用现有 CAN 通信、运动学、电机控制代码（不改一行现有代码）。

---

## 当前架构

```
hls_canopen/
├── Cargo.toml
├── src/
│   ├── lib.rs                    ← pub mod 声明
│   ├── main.rs                   ← WebSocket cmd_vel (binary: hls_canopen)
│   ├── can_driver.rs             ← CAN 抽象层
│   ├── can_setup.rs              ← ip link 配置 can0
│   ├── kinematics.rs             ← 差速运动学 + 位姿积分
│   ├── motor.rs                  ← 电机控制 (NMT, PV mode, enable/disable)
│   └── sdo_client.rs             ← CANopen SDO 协议
│   └── bin/
│       └── hls_cli.rs            ← CLI REPL (binary: hls_cli)
└── .hermes/plans/
```

**硬件参数**（固定常数，ROS2 节点通过 clap 参数可覆盖）：
- 轮径: 0.0845 m (ø169 mm)
- 轮距: 0.45 m
- 右轮反向: true
- CAN: can0, 1 Mbps
- 左节点 0x01，右节点 0x02
- 控制周期: 50 ms (20 Hz)
- 看门狗: 1s

## ROS2 主题设计

### 订阅

| 主题 | 类型 | 说明 |
|------|------|------|
| `/cmd_vel` | `geometry_msgs/msg/Twist` | 速度指令 `.linear.x`(m/s), `.angular.z`(rad/s) |

### 发布

| 主题 | 类型 | 频率 | 说明 |
|------|------|------|------|
| `/odom` | `nav_msgs/msg/Odometry` | 20 Hz | 里程计：位姿 (x,y,theta) + 速度 (v,w) |
| `/tf` | `tf2_msgs/msg/TFMessage` | 20 Hz | `odom` → `base_link` 变换 |

### TF 树

```
odom (fixed frame) ──→ base_link (机器人本体)
```

### 帧 ID 约定

| 字段 | 值 |
|------|-----|
| `odom.header.frame_id` | `"odom"` |
| `odom.child_frame_id` | `"base_link"` |
| `tf.transform.header.frame_id` | `"odom"` |
| `tf.transform.child_frame_id` | `"base_link"` |

### 协方差

- 位姿协方差: 初始设为 `[0.001; 36]`（基于编码器的轮式里程计，精度较好）
- 速度协方差: 初始设为 `[0.01; 36]`
- 可通过 `--pose-covariance` / `--twist-covariance` 参数覆盖

## 实现方案

### 新建文件

| 文件 | 说明 |
|------|------|
| `src/bin/hls_ros2.rs` | ROS2 桥接节点（新 binary `hls_ros2`） |

### 改动文件

| 文件 | 改动 |
|------|------|
| `Cargo.toml` | 添加 `r2r` 依赖 + `[[bin]]` 声明 `hls_ros2` |

### 核心逻辑 (`hls_ros2.rs`)

结构上复用 `main.rs` 的 WebSocket 模式，把 WebSocket 替换为 r2r 主题：

```
初始化:
  1. clap 解析参数 (--can-iface, --wheel-radius, --wheel-base 等)
  2. can_setup::setup_can(can_iface, 1_000_000)
  3. SocketCanBus::open + SdoClient + Motor
  4. NMT start + set_mode(PV) + set_accel + set_velocity(0) + enable
  5. 创建 r2r Context + Node
  6. 创建 /cmd_vel subscriber
  7. 创建 /odom publisher
  8. 创建 /tf publisher

控制循环 (20 Hz):
  1. spin_once(Duration::from_millis(1))
  2. 如果有新的 cmd_vel：通过 DiffDrive::inverse() 转 RPM → set_velocity()
  3. 更新 last_cmd 时间戳
  4. 看门狗检查：超时 1s → disable()
  5. 读实际 RPM (SDO 0x606C) → wheel_velocities() → integrate_pose()
  6. CAN 连续错误 >= 10 → 退出（systemd Restart=always 拉起）
  7. 发布 /odom (nav_msgs/Odometry): pose = {x,y,theta}, twist = {v,w}
  8. 发布 /tf (tf2_msgs/TFMessage): odom → base_link
```

### cmd_vel 回调

```rust
// ROS2 callback style (r2r)
// 收到 Twist 消息 → DiffDrive::inverse(linear.x, angular.z) → set_velocity()
// 支持同时多连接（参考 main.rs 设计）
```

### odom 消息构建

```rust
// pose.position = {x, y, 0}
// pose.orientation = Quaternion from yaw (theta)
//    q.z = sin(theta/2)
//    q.w = cos(theta/2)
// twist.linear.x = v (m/s)
// twist.angular.z = w (rad/s)
```

### 错误处理（继承 main.rs 设计）

- CAN 读取错误 → 注入 CAN down 状态，冻结位姿积分
- 连续 10 次错误 → exit（systemd 拉起）
- set_velocity 失败 → disable + break 内循环
- 退出前 safety_disable

## 分步计划

### Step 1: 添加依赖和 binary 声明

修改 `Cargo.toml`：
```toml
[[bin]]
name = "hls_ros2"
path = "src/bin/hls_ros2.rs"

[dependencies]
# ... 现有 ...
r2r = "0.9.5"
clap = { version = "4", features = ["derive"] }
```

### Step 2: 编写 hls_ros2.rs

核心节点，~350 行：
- clap 参数定义
- 初始化流程（CAN → NMT → mode → enable）
- r2r 节点 + subscriber + publishers
- 控制循环（cmd_vel → CAN → odom → tf）

### Step 3: 编译

```bash
zsh -c 'source ~/.zshrc; cd ~/ros2_ws/src/hls_canopen && cargo build --bin hls_ros2'
```

### Step 4: 创建启动脚本

`~/hls_ros2.sh` — 类似 `~/keli_lidar.sh` 的风格

```bash
#!/bin/zsh
source ~/.zshrc
exec ~/ros2_ws/src/hls_canopen/target/debug/hls_ros2 "$@"
```

### Step 5: 验证

```bash
# 终端 1: 启动驱动
zsh -c 'source ~/.zshrc; ~/ros2_ws/src/hls_canopen/target/debug/hls_ros2'

# 终端 2: 发指令
zsh -c 'source ~/.zshrc; ros2 topic pub /cmd_vel geometry_msgs/Twist "{linear: {x: 0.5}, angular: {z: 0.0}}" --once'

# 终端 3: 看 odom
zsh -c 'source ~/.zshrc; ros2 topic echo /odom --once'

# 终端 4: 看 tf
zsh -c 'source ~/.zshrc; ros2 topic echo /tf --once'

# 终端 5: 看 tf 树
zsh -c 'source ~/.zshrc; ros2 run tf2_tools view_frames'
```

### Step 6: Git 提交推送

```bash
cd ~/ros2_ws/src/hls_canopen
git add -A
git commit -m "feat: hls_ros2 — 差速底盘 ROS2 桥接 (r2r, cmd_vel→odom/tf)"
```

## 对应关系：WebSocket 版本 → ROS2 版本

| WebSocket 组件 | ROS2 等效 |
|---------------|-----------|
| `ws://0.0.0.0:9090` | r2r Node |
| JSON `{"linear": x, "angular": y}` 输入 | `/cmd_vel` subscriber (Twist) |
| JSON odom 输出 | `/odom` publisher (Odometry) + `/tf` |
| 20 Hz 控制循环 | 不变，保持 20 Hz |
| 位姿积分 (x,y,theta) | 不变 → 填入 Odometry.pose + TF |
| CAN 错误处理 | 不变 |
| 看门狗 | 不变 |

## 风险和待确认

1. **r2r 多 publisher 支持** — 需要同时 publish `/odom` 和 `/tf`，r2r 支持多个 publisher，没问题
2. **tf2_msgs/TFMessage** — r2r 中对应的类型名需要确认（可能是 `tf2_msgs::msg::TFMessage`）。如果 r2r 不自带，可能需要手动生成
3. **/cmd_vel 调用频率** — 当前设计是 20 Hz polling + spin_once，cmd_vel 回调会设置 shared state，控制循环读 state。这跟 WebSocket 版按消息驱动的模式不同，需要调整为 ROS2 惯用模式
4. **`integrate_pose` 已用 Euler 积分** — 短时间没问题，长时间运行会漂移，后续可改为 Runge-Kutta 或融合 IMU
