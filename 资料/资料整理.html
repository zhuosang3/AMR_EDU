EXP2 无人配送车 — 项目资料大全

最后更新: 2026-06-11
机器人: Orange Pi 5 Plus / Armbian Noble / ROS2 Jazzy
仓库组织: https://cnb.cool/zhazhatuan/edu/ 本次更新: 位置差分里程计 + EKF 融合 + 0x6063/0x6064 坑 + 轮距标定 0.39

一、系统架构全貌

┌─ 硬件驱动层 (~/) ────────────┐     ┌─ ROS2 桥接层 (ros2_ws/src/) ─┐
│                               │     │                               │
│  imu_driver ──WS:8765──→  imu_bridge ──→ /imu                    │
│  hls_canopen ─WS:9090─→  canopen_ros2 ──→ /odom, /tf            │
│                                  └─────── ← /cmd_vel              │
│  [keli_lidar ──UDP:2112──→  /scan]       (纯ROS2节点)            │
│  [exp2_description ──────→  /tf_static, /robot_description]      │
│  [exp2_nav ──────────────→  Nav2 导航栈]                          │
└───────────────────────────────────────────────────────────────────┘

分层原则: 硬件驱动层无 ROS 依赖（纯 Rust cargo build），ROS2 桥接层通过 WebSocket 与驱动层通信。驱动可脱离 ROS 独立测试。

五、系统架构（当前 2026-06-11）

┌─ 硬件驱动层 (~/) ────────────┐     ┌─ ROS2 桥接层 (ros2_ws/src/) ──────────┐
│                               │     │                                        │
│  imu_driver ──WS:8765──→  imu_bridge ──→ /imu (100Hz, gyro cov 0.001)     │
│  hls_canopen ─WS:9090─→  canopen_ros2 ──→ /odom (20Hz, 不发TF)            │
│                           (--publish-tf false) │                             │
│                                      ┌─────────┘                             │
│                              ┌───────┴──────────┐                            │
│                              │ ekf_filter_node   │ ← robot_localization      │
│                              │ (world_frame=odom)│                            │
│                              └───────┬──────────┘                            │
│                                      └──→ /odometry/filtered (20Hz)          │
│                                      └──→ TF odom→base_link (独家广播)       │
│                                                                              │
│  [keli_lidar ──UDP:2112──→  /scan]        (纯ROS2节点)                       │
│  [exp2_description ──────→  /tf_static, /robot_description]                  │
│  [exp2_nav ──────────────→  Nav2 导航栈]                                     │
│      - bt_navigator.odom_topic = /odometry/filtered                          │
│      - velocity_smoother.odom_topic = /odometry/filtered                     │
│      - local_costmap.global_frame = odom                                     │
└──────────────────────────────────────────────────────────────────────────────┘

EKF 融合策略: odom 取 [vx, vyaw] + IMU 取 yaw 角速度 → /odometry/filtered. 不融合 IMU 线加速度（双重积分噪声爆炸, 轮式机器人不用）。 AMCL 仍发布 map→odom（world_frame=odom）。

八、启动流程

8.1 导航模式（已有地图 → AMCL 定位）

# 前提：PM2 所有驱动进程 running
ros2 launch exp2_nav navigation.launch.py

此 launch 文件内部：

  • 启动 localization_launch.py（AMCL + map_server，加载 /home/expressone/maps/my_map.yaml）
  • 启动 navigation_launch.py（controller/planner/behavior/costmap 等节点）

8.2 建图模式（SLAM）

ros2 launch exp2_nav mapping.launch.py

此 launch 文件内部：

  • 启动 navigation_launch.py
  • 启动 async_slam_toolbox_node
  • 启动 map_saver_server（方便随时保存地图）

保存地图：

ros2 run nav2_map_server map_saver_cli -f ~/maps/my_map

8.3 前置检查

启动 Nav2 前确认：

ros2 topic list | grep -E "/scan|/odom|/tf|/cmd_vel"

五项必须全部在线：/scan, /odom, /tf, /tf_static, /cmd_vel 有订阅者。

十四、2026-06-11 旋转漂移修复 (关键改动)

14.1 根因

AGV 稍微旋转就漂移、直线还好。分析：原里程计用 0x606C RPM 反馈积分 — 0x606C 单位 0.1 rpm，整数截断。直线 0.3m/s≈34RPM，0.5RPM 误差仅 1.5%；但原地旋转差速仅 ±3.8RPM， 0.5RPM 误差≈13%，角度迅速发散。次要因素：后万向轮打滑、固定 DT 0.05s 欠积分、轮距未标定。

14.2 治本方案：位置差分里程计 (0x6063 → 5600 counts/rev)

改动 文件 说明
数据源切 0x6063 sdo_client.rs 0x6064 恒为 0 (DS20270DA 不填), 用 0x6063
编码器分辨率 电机型号 10530NA1550F20 “A” 型 = 1400 线光编 × 4 倍频 = 5600 counts/rev
单位修正 kinematics.rs RAD_PER_COUNT = TAU/5600 (电机手册命名规则+实测验证)
SDO 全部不暴露 0x608F.1/0x2006/0x2007/0x6091 均返回 abort 或超时, 只能靠电机型号反查
防计数器回绕 main.rs wrapping_sub 处理 i32 溢出
中点法积分 kinematics.rs RK2 替代欧拉
断线冻结 main.rs CAN 故障冻结 pose + 重置计数基线

14.3 兜底方案：robot_localization EKF

组件 说明
融合信号 odom 取 [vx, vyaw] + IMU 仅 yaw 角速度
不融合 IMU 加速度 双重积分噪声爆炸, 轮式机器人不用
TF 切换 EKF 独家广播 odom→base_link; canopen_ros2 –publish-tf false
Nav2 衔接 bt_navigator + velocity_smoother odom_topic → /odometry/filtered

十七、其他常用脚本命令 && 注意事项

  • 关于如何使用终端工具Terminator，可以上网搜索，很简单易学（ctrl+alt+t是启动快捷键，ctrl+shift+o是分屏，ctrl+shift+w是关闭）
  • ./disable.zsh 电机失能（推动之前一定要做；另外不要推得太快否则会有电流产生）
  • ./enable.zsh 电机使能（注：建图、导航之前一定要让电机使能否则走不了）
  • ros2 run teleop_twist_keyboard teleop_twist_keyboard 用键盘控制小车走（线速度和角速度不建议太高，0.2左右；并且调整速度时要按K键，否则也会移动），具体控制方式见命令
  • 要根据实际导航场景来调整算法参数如膨胀率，过大的话遇到窄路可能失灵，过小的话遇到宽路会出现靠边走的情况，参数文件位于/home/expressone/ros2_ws/src/exp2_nav/config下的nav2_params.yaml中，建议使用vscode打开，找到global_costmap下的inflation_radius（位于第206行）并且修改
  • 建图快捷脚本：登录ubuntu后直接运行mapping.zsh，启动建图，并且等待不再有新的蓝色字体跳出后，再执行rviz2 -d ~/exp2.rviz，可观察实时雷达图像。注意要缓慢推动，不要有大幅度晃动；尽量避免扫描重复位置否则图像会出现割裂。
  • 保存地图直接运行savemap.zsh保存，注意有一定概率保存失败，最好打开ros2_ws/src/exp2_nav/config/maps/下的地图文件查看，如果不符合预期说明未保存成功，需要重试，一般两次都能成功。
  • 重定位的话，直接在rviz2中手动选择起点即可，注意一定要准确，否则可能导航失败，因为系统认为雷达实际扫描结果与地图差别过大。
  • 上次导航可能有残留进程和文件，运行./stop_mapping.zsh脚本来清理，之后可以下一次运行
  • 尽量先在右上角power off之后再关掉电源
  • 充电时先插车上的，再插插座
